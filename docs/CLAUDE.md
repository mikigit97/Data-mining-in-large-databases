## Database Schema

**Database file:** `baby_names.db` (SQLite 3, ~395 MB)

```sql
CREATE TABLE national_names (
    Id      INTEGER PRIMARY KEY,
    Name    TEXT    NOT NULL,
    Year    INTEGER NOT NULL,   -- 1880–2014
    Gender  TEXT    NOT NULL,   -- 'M' or 'F'
    Count   INTEGER NOT NULL
);

```

**Indexes on `national_names`:**
- `idx_nat_name` on `(Name)`
- `idx_nat_year` on `(Year)`
- `idx_nat_name_year` on `(Name, Year)` — composite/covering

## App Architecture (`app.py`)

- **Framework:** Streamlit
- **Charts:** Plotly Express
- **DB connection:** `@st.cache_resource` singleton via `get_conn()`
- **Three tabs:**
  - Tab A — Name Popularity Over Time (raw count / % of births toggle)
  - Tab B — Custom SQL Query Panel (SELECT-only safety check, 5 example queries, auto-chart)
  - Tab C — Name Insights (diversity over time + peak decade finder)
- **Run locally:** `python -m streamlit run task1/app.py` from the repo root (or `cd task1 && streamlit run app.py`)
- **Deploy:** push to GitHub → connect repo to Streamlit Cloud → it reads `requirements.txt` automatically

## Key Design Decisions

- Data loaded in **50 000-row chunks** (pandas `chunksize`) to handle the 148 MB StateNames.csv without memory issues.
- `PRAGMA synchronous = OFF` + `journal_mode = MEMORY` used during bulk insert for ~10× speedup, then restored to safe defaults.
- DB is **deleted and recreated** at the top of the notebook to ensure full reproducibility.
- SQL safety: checks the **first non-comment keyword** (skips leading `--` lines) and blocks anything that is not `SELECT`.
- Relative popularity uses a **left join** against a pre-aggregated totals table (135 rows) — fast and safe.

## Bugs Found & Fixed

| Subtask | Bug | Fix |
|---|---|---|
| 1.2 | SQL safety check bypassed by leading `--` comment lines | Loop over lines, skip `--` prefixed ones before reading first keyword |
| 1.2 | `df.merge(totals, on="Year")` inner join silently drops rows | Changed to `how="left"` + `.replace(0, 1)` on Total column |
| 1.2 | Free axis selectors allowed meaningless combos (`x=Name, y=Year`, `color=Year`) | Bar: Y=fixed measure, color=Gender only; Line: X=Year fixed, color=Name or Name(Gender) |
| 3.1 | `Optional(str)` rejects `None` — `ValueError: Attribute Pokemon.type2 cannot be set to None` | Changed to `Optional(str, nullable=True)` to store SQL NULL for single-type Pokémon |
| 3.1 | All 800 Pokémon stored as `legendary=True` — `bool(NaN)` is `True` | `df['legendary'].map({'True':True,'False':False})` failed because pandas already read the column as `bool` dtype (string keys don't match); removed the `.map()` call entirely |
